# 2025安全知识推广普及益智游戏策划
1. 1v1卡牌对战
双方交替出击，p1打出攻击卡后p2能使用防御卡减少所受伤害。
每名玩家回合内只能打出两张攻击卡，但先手玩家首回合只能打出一张攻击卡
每回合抽两张，抽牌阶段如果牌库没牌判负
打出的牌不重新洗入牌库

攻击卡
属性	示例
名称	SQL 注入
类型	攻击卡
攻击力 3
一级tag BLOCK_ALL, BLOCK_TYPE_X
二级tag SOCIAL_ENGINEERING WEB_ATTACK SYSTEM_EXPLOIT
三级tag CHAINABLE PERSISTENT
连锁目标  与哪种二级tag同时打出会触发
连锁效果  [攻击效果池]
持续时间  n回合
说明	常见于未过滤的用户输入

防御卡
属性 示例
名称 防火墙
防御力 2
特定强化 针对哪种二级攻击tag有防御强化
强化效果 从池中抽选

策略卡
属性 示例
名称 漏洞修复补丁
效果 从池中抽选

✅ 一级分类：攻击目标机制
标记	功能说明
DIRECT_ONLY	【普通】只能攻击部署中的防御卡，不能直接攻击主机（默认）
SPLASH	【穿透】穿透攻击，攻击防御卡的同时对主机造成额外溢出伤害例如社工、APT

✅ 二级分类：攻击类型（可作为卡组构建或卡效限制）
标记	安全术语映射	示例卡牌
SOCIAL_ENGINEERING	【社会工程学】	钓鱼邮件、假冒电话
WEB_ATTACK	【web攻击】	SQL注入、XSS
SYSTEM_EXPLOIT	【系统漏洞】操作系统漏洞	RDP入侵、提权攻击

✅ 三类标签：辅助标签（效果触发条件/反制点）
标记	说明
CHAINABLE	可与其他攻击组合造成更强效果 、 二级分类中两两组合

🟦 特殊卡 / 工具卡（不参与战斗，但产生buff/debuff）
标记 效果
addHealth 恢复一个防御卡或水晶生命值
enemyDebuff 对方下回每张牌攻击伤害减少
addSelf 下回合 伤害增加

## 效果池

|名称| 参数|描述|
|-|-|-|
|c_1|n 增加值|增加攻击卡伤害一回合|
|d_1|n 增加值|增加防御卡防御力|

|s_1|n 回血|自己回复n点穴|
|s_2|n 抽牌|立刻抽n张牌|
|s_4|n|自己本回合攻击卡增加n点战力|
|s_5|n|自己防御卡增加n点防御一回合|
|s_6|n|对方玩家攻击减少n，持续一回合|
